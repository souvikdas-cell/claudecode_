Part 1 — Executive Summary
For VPs & Product

What the Widget Does

The FinX Discovery Widget surfaces the most significant option market moves happening right now — before traders have to dig through chains, scanners, or news feeds. It is a live, signal-first surface embedded in the StrikeX platform that tells a trader what is happening, how strong the move is, and whether they should act now or wait.

Two Modules at a Glance

ModuleWhat It ScansWho It's ForKey Output
OI Flow Open Interest changes in NIFTY, SENSEX, and F&O stocks Directional traders — spotting fresh positions by big players Strike, OI change, signal type (Bull / Bear / Cover / Unwind), Status badge
IV Spikes Implied Volatility jumping above 30-day average after big price moves Option writers — selling premium after fear spikes Best strike to write, premium, margin required, yield %

OI Signal Types — Plain English

Every OI Flow signal is one of four types. They are deterministic — derived from combining the direction of OI change and price change. No human judgment involved.

Bull Flow
OI going up + Price going up
New long positions being added. Smart money is buying calls (or the underlying). Bullish signal.
Bear Pressure
OI going up + Price going down
New short positions being added. Sellers are piling in. Bearish signal.
Short Cover
OI going down + Price going up
Existing shorts are being forced to cover. Price rises as shorts buy back. Bullish (reactive).
Unwinding
OI going down + Price going down
Existing longs are exiting. Loss of conviction. Bearish (weakening).

Status — The Action Badge

The Status column tells the trader what to do. It collapses three inputs — signal direction, how fresh the signal is, and volume confirmation — into a single actionable badge. No raw data interpretation needed.

BadgeColorMeaningTrader Action
ENTRYGreenFresh signal, confirmed by high volumeAct now — opportunity is live
IN PLAYBlueSignal is live, position still validRide it — signal ongoing
WATCHAmberSignal present but not yet confirmedWait for volume or price follow-through
FADINGGreySignal is old — momentum may be goneDon't chase — wait for next signal
💡

The sub-line below every Status badge shows signal age (e.g., "4 min ago") — so traders always know if they are looking at a real-time alert or something from an hour ago.

How Status is Decided

Three inputs are combined:

  • Direction — is the signal bullish (Bull / Cover) or bearish (Bear / Unwind)?
  • Age — how many minutes ago was the OI move detected? Thresholds differ by time window.
  • Volume — is today's volume ≥ 4× the daily average? If yes, the signal is confirmed.
DirectionSignal AgeVolume→ Status
BullishFRESH≥ 4× averageENTRY
BullishFRESH< 4× averageWATCH
BullishACTIVEAnyIN PLAY
BullishAGINGAnyFADING
BearishFRESHAnyWATCH
BearishACTIVE or AGINGAnyFADING
⚠️

Why bearish signals never show ENTRY: Bear and Unwind signals are intentionally held at WATCH/FADING — acting on the short side requires more confirmation. This is a deliberate product decision to reduce false bearish entries.

Signal Story — Derived Text

The "Signal Story" column text is derived from two fields: sig and vol. High volume = confirmation. Low volume = early/forming signal.

sigvol condition→ Text
"bull"vol ≥ 5×Strong Bull Flow
"bull"vol < 5×Bull Flow Building
"cover"vol ≥ 5×Short Squeeze
"cover"vol < 5×Shorts Retreating
"bear"vol ≥ 5×Bear Pressure Rising
"bear"vol < 5×Bear Pressure
"unwind"vol ≥ 4×Longs Unwinding
"unwind"vol < 4×OI Draining
 
function getStory(sig, vol) {
  var v = parseFloat(vol);         // vol comes in as "3.8×"
  var map = {
    bull:   v >= 5 ? 'Strong Bull Flow'      : 'Bull Flow Building',
    cover:  v >= 5 ? 'Short Squeeze'         : 'Shorts Retreating',
    bear:   v >= 5 ? 'Bear Pressure Rising'  : 'Bear Pressure',
    unwind: v >= 4 ? 'Longs Unwinding'       : 'OI Draining'
  };
  return map[sig] || 'Signal Forming';
}

Status Derivation — Full Logic

Status is computed from sig, ts (signal timestamp), vol, and the active time window.

Step 1 — Signal Direction

isBullish = (sig === 'bull' || sig === 'cover')

Step 2 — Signal Age Thresholds (by active time window)

Time WindowFRESH (≤)ACTIVE (range)AGING (>)
5 MIN2 min3–4 min4 min
15 MIN5 min6–12 min12 min
1 HOUR15 min16–40 min40 min
TODAY30 min31–120 min120 min
Stocks20 min21–60 min60 min
 
var thresholds = {
  '5m':  [2,  4],
  '15m': [5,  12],
  '1h':  [15, 40],
  '1d':  [30, 120],
  'stk': [20, 60]
};
var age       = NOW_TS - row.ts;     // minutes since signal was detected
var [fresh, active] = thresholds[window];
var ageClass  = age <= fresh ? 'FRESH' : age <= active ? 'ACTIVE' : 'AGING';

Step 3 — Volume Confirmation

isConfirmed = parseFloat(row.vol) >= 4;   // vol field: "3.8×" → 3.8

Step 4 — Final Decision

if (isBullish) {
  if (ageClass === 'FRESH' && isConfirmed)   → 'ENTRY'   // green
  if (ageClass === 'FRESH' && !isConfirmed)  → 'WATCH'   // amber
  if (ageClass === 'ACTIVE')               → 'IN PLAY' // blue
  if (ageClass === 'AGING')                → 'FADING'  // grey
} else {                                             // bearish
  if (ageClass === 'FRESH')                → 'WATCH'   // amber
  else                                    → 'FADING'  // grey
}

Strike Context Formula

For Index signals, a context label is derived showing how far the signal strike is from the current ATM.

var diff = Math.round((stk - ATM[idx]) / STEP[idx]);

// CE (Call)
diff === 0                  → "At ATM"
diff === +1 || diff === -1  → "ATM±1 · Near ATM"
diff > 0  (CE OTM)         → "ATM+N OTM"   // e.g. "ATM+3 OTM"
diff < 0  (CE ITM)         → "ATM−N ITM"

// PE (Put)
diff < 0  (PE OTM)         → "ATM−N OTM"
diff > 0  (PE ITM)         → "ATM+N ITM"
📌

Example: NIFTY ATM = 24850, Strike = 25000 → diff = +3 → label = "ATM+3 OTM"

Time Frame Windows

OI Flow data is grouped by time window. Each window uses different freshness thresholds for the Status calculation. The same signal can appear in multiple windows — older windows show more signals but with lower freshness.

LabelKeyWindowUse Case
5 MIN'5m'Last 5 minutesScalping, ultra-fresh intraday entries
15 MIN'15m'Last 15 minutesShort-term momentum confirmation
1 HOUR'1h'Last 60 minutesTrend-building signals
TODAY'1d'Full session (9:15–15:30)All signals from the current trading day
